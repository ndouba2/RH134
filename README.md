
## Configuration avancée du système de fichiers

La configuration avancée du système de fichiers sous Red Hat Enterprise Linux (RHEL) est l'un des sujets couverts dans la formation RH134. Voici un aperçu des sujets abordés :

Configuration de LVM : Logical Volume Manager (LVM) est un outil de gestion de stockage qui permet de créer des volumes logiques en utilisant des volumes physiques. LVM permet la gestion flexible de l'espace disque en permettant la création de volumes logiques qui peuvent s'étendre sur plusieurs disques physiques. La formation RH134 couvre la création de volumes physiques, de groupes de volumes logiques et de volumes logiques.

Configuration du RAID : Redundant Array of Independent Disks (RAID) est une technologie qui permet de regrouper plusieurs disques physiques en un seul volume logique pour améliorer la performance et la tolérance aux pannes. La formation RH134 couvre la configuration du RAID matériel et logiciel, y compris la création de volumes RAID 0, 1 et 5.

Configuration du stockage réseau : La configuration du stockage réseau est une technique qui permet d'accéder à des volumes de stockage à partir d'un serveur de stockage distant via un réseau. La formation RH134 couvre la configuration de iSCSI (Internet Small Computer System Interface), qui permet l'accès aux volumes de stockage à partir d'un serveur de stockage iSCSI distant.

Configuration de NFS : Network File System (NFS) est un protocole de partage de fichiers qui permet de partager des fichiers et des répertoires entre des systèmes sur un réseau. La formation RH134 couvre la configuration de NFS pour partager des fichiers entre les systèmes Linux.

Configuration de quotas de disque : Les quotas de disque permettent de limiter l'espace disque qu'un utilisateur ou un groupe peut utiliser. La formation RH134 couvre la configuration des quotas de disque pour contrôler l'utilisation de l'espace disque sur le système de fichiers.

En résumé, la formation RH134 couvre la configuration avancée du système de fichiers sous RHEL en utilisant des techniques telles que LVM, RAID, stockage réseau, NFS et quotas de disque. Les participants apprendront à configurer ces technologies pour améliorer la performance, la disponibilité et la sécurité du système de fichiers.


## Contrôle d'accès aux fichiers avec les permissions et les acls

Le contrôle d'accès aux fichiers avec les permissions et les ACLs (Access Control Lists) est un sujet important dans la formation RH134. Voici un aperçu des sujets couverts :

Permissions de fichiers : Les permissions de fichiers permettent de contrôler l'accès aux fichiers en définissant des droits pour les propriétaires, les groupes et les autres utilisateurs. La formation RH134 couvre les types de permissions, les commandes pour modifier les permissions et les combinaisons de permissions pour différents scénarios d'utilisation.

Permissions de répertoires : Les permissions de répertoires permettent de contrôler l'accès aux fichiers dans un répertoire en définissant des droits pour les propriétaires, les groupes et les autres utilisateurs. La formation RH134 couvre les types de permissions de répertoire, les commandes pour modifier les permissions et les combinaisons de permissions pour différents scénarios d'utilisation.

Access Control Lists (ACLs) : Les ACLs permettent de définir des permissions plus granulaires pour les fichiers et les répertoires, en permettant la définition de permissions pour des utilisateurs et des groupes spécifiques. La formation RH134 couvre la définition des ACLs, les commandes pour modifier les ACLs et les combinaisons de permissions pour différents scénarios d'utilisation.

Contexte de sécurité SELinux : Le contexte de sécurité SELinux est une technique de sécurité qui permet de contrôler l'accès aux ressources système en fonction de règles de sécurité définies. La formation RH134 couvre la configuration de SELinux pour utiliser les contextes de sécurité et la modification des contextes de sécurité pour les fichiers et les répertoires.

En résumé, la formation RH134 couvre le contrôle d'accès aux fichiers avec les permissions et les ACLs en utilisant des techniques telles que les permissions de fichiers, les permissions de répertoires, les ACLs et le contexte de sécurité SELinux. Les participants apprendront à configurer ces techniques pour assurer la sécurité des fichiers et des répertoires sur le système.
